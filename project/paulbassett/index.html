모바일 주문 어플 "폴바셋"
Mobile UI/UX Redesign

팀 프로젝트로 누구나 쉽게 폴바셋 앱을 사용할 수 있도록 리디자인 하였습니다.

기간 : 22.04.29 - 22.06.28

팀원 : 열심히하는 조 (이주현, 김나연, 서리미)

01. 프로젝트 소개.

  • 선정 앱

    폴바셋 ( 모바일 주문 앱 )

  • 선정 이유

    요즘 비대면 주문 앱이 많이 증가하고 있습니다. 하지만 폴바셋 앱은 다른 앱과 비교했을 때 시각적, 기능적으로 부족해 보이는 부분이 보였고 이를 개선하고자 선정하게 되었습니다.

02. 진행과정.

  • 개선하고자 하는 앱 선정
  • 경쟁사 앱 데스크리서치
  • 선정한 앱 데스크리서치
  • UX 비전 설정
  • 페르소나 설정 및 시나리오 작성
  • 스타일가이드 제작
  • 와이어프레임 제작
  • 프로토타입 제작

03. 담당 업무.

  • 경쟁사(투썸 플레이스) 애플리케이션 분석
  • 선정 애플리케이션 분석
  • 사용자들을 위한 UX Vision 의견 제안
  • UX 디자인 키워드 제안
  • 사용자 여정지도 의견 제안 및 내용 정리
  • 와이어 프레임 구성 의견 제시 및 제작
  • 디자인 레퍼런스 자료수집 및 의견 제시
  • 프로토타입 제작

04. 프로토타입.

UX vision : 사용자 누구나 손쉽게 사용할 수 있도록 "누구나 쉽게" 로 정했습니다.

05. 프로젝트 결과.

  • 단조롭고 올드한 디자인에서 트렌드에 맞춘 새로운 디자인 으로 개선했습니다.
  • 직관성이 떨어지거나 터치를 실수 할 수 있는 부분을 사용자의 편의성을 높이는 디자인으로 개선했습니다.
  • 중복되는 카테고리 문제로 불필요한 카테고리를 최소화 시켰습니다.

06. 프로젝트 평가.

  • 부족한 점

    • 처음해본 디자인 : UI/UX 디자인을 처음 접해보아 다양한 레퍼런스를 찾았음에도 실제로 구성하고 적용시켜보니 폰트 사이즈, 강약 조절 등 디자인을 부족한 부분과 어색한 부분이 많았습니다.
    • 비대면의 의사소통 : 비대면으로 원활한 의사소통이 어려웠고 , 그로 인해 시간이 오래걸린점이 아쉬웠습니다.
    • 주어진 시간 : 주어진 시간안에 디자인과 기능을 구현시켜야 하니 많은 페이지와 기능을 꼼꼼히 구성하지 못해 아쉬움이 있습니다.

  • 잘한 점

    • 팀원과의 소통 : 프로젝트에 필요한 작업을 팀원들과 공유하고 다양한 의견을 제시하며 소통했습니다. 그로 인해 주어진 시간안에 프로젝트를 마무리 할 수 있었습니다
    • 사용자의 입장에서 완성 : 버튼 크기나, 불편한 부분, 수정 되었으면 좋겠는 부분을 데스크 리서치를 통해 파악하고 앱을 리디자인 하였습니다.

  • 배운 점

    • 앱 디자인의 세부 과정 : 어떠한 과정과 흐름으로 앱이 디자인되는지 배웠습니다.
    • 팀원과의 소통방법 : 다양한 의견을 제시하고 조율하면서 매끄럽게 소통하는 방법을 배웠습니다.
    • 디자인 툴 사용법 : Adobe XD, Adobe illustrator을 직접 사용하여 사용법을 익혔습니다.

07. Review.

  • 처음해본 UX/UI 디자인, 프로젝트가 처음이라 걱정이 앞섰습니다. 하지만 팀 프로젝트다 보니 내가 부족한 부분은 다른 팀원들이 채워줘서 끝까지 포기하지않고 잘 마무리 할 수 있었던 것 같습니다. 또한 내가 다양한 의견을 제시 했을 때 긍정적인 의견을 줘서 뿌듯했고 , 더 다양한 의견을 제시 할려고 노력했습니다. 또한 발표를 통해 수정해야할 부분과 잘한 부분을 파악 할 수 있어서 좋았습니다.

    100%로 만족하는 결과는 아니지만 모두 노력해 최선의 결과물을 냈다고 생각합니다. 이번 팀 프로젝트는 저를 한층 더 발전시켜주었고, 다른 프로젝트에서도 지금의 경험을 바탕으로 좋은 결과물을 낼 수 있을 것 같습니다.
