LANCOME

RESPONSIVE UI RENEWAL PROJECT

01. 프로젝트 소개.

  • 프로젝트 명

    폴바셋 애플리케이션 리디자인

  • 기간

    2022.04.29 ~ 2022.06.28

  • 팀원

    열심히하는 조 ( 이주현, 김나연, 서리미 )

  • 선정 앱

    폴바셋 ( 모바일 주문 앱 )

  • 선정 이유

    다른 카페 앱과 비교했을 때 시각적, 기능적으로 부족해 보이는 부분이 보였고 이를 개선하고자 선정하게 되었습니다.

02. 진행과정.

  • 개선하고자 하는 앱 선정
  • 경쟁사 앱 데스크리서치
  • 선정한 앱 데스크리서치
  • UX 비전 설정
  • 페르소나 설정 및 시나리오 작성
  • 스타일가이드 제작
  • 와이어프레임 제작
  • 프로토타입 제작

03. 담당 업무.

  • 경쟁사(투썸 플레이스) 애플리케이션 분석
  • 선정 애플리케이션 분석
  • 사용자들을 위한 UX Vision 의견 제안
  • UX 디자인 키워드 제안
  • 사용자 여정지도 의견 제안 및 내용 정리
  • 와이어 프레임 구성 의견 제시 및 제작
  • 디자인 레퍼런스 자료수집 및 의견 제시
  • 프로토타입 제작

04. 프로토타입.

05. 프로젝트 결과.

  • UI/UX 디자인을 처음 접해보아 다양한 레퍼런스를 찾았음에도 디자인을 실제로 구성하고 적용시켜보니 폰트 사이즈, 강약 조절 등 부족한 부분과 어색한 부분이 많았습니다.

  • 비대면으로 원활한 의사소통이 어려웠고 , 그로 인해 시간이 오래걸린점이 아쉬웠습니다.

06. 프로젝트 평가.

  • 부족한 점

    • 처음해본 디자인 : UI/UX 디자인을 처음 접해보아 다양한 레퍼런스를 찾았음에도 폰트 사이즈, 강약 조절 등 디자인을 실제로 구성하고 적용시켜보니 부족한 부분과 어색한 부분이 많았습니다.
    • 비대면의 의사소통 : 비대면으로 원활한 의사소통이 어려웠고 , 그로 인해 시간이 오래걸린점이 아쉬웠습니다.

  • 잘한 점

    • 팀원과의 소통 : 프로젝트에 필요한 작업을 팀원들과 공유하고 다양한 의견을 제시하며 소통했습니다. 그로 인해 주어진 시간안에 프로젝트를 마무리 할 수 있었습니다
    • 사용자의 입장에서 완성 : 버튼 크기나, 불편한 부분, 수정 되었으면 좋겠는 부분을 데스크 리서치를 통해 파악하고 앱을 리디자인 하였습니다.

  • 배운 점

    • 앱 디자인의 세부 과정 : 어떠한 과정과 흐름으로 앱이 디자인되는지 배웠습니다.
    • 팀원과의 소통방법 : 다양한 의견을 제시하고 조율하면서 매끄럽게 소통하는 방법을 배웠습니다.
    • 디자인 툴 사용법 : Adobe XD, Adobe illustrator을 직접 사용하여 사용법을 익혔습니다.

07. Review.

  • # 내 첫 프로젝트

    디자인부터 퍼블리싱까지 스스로 모든것을 구현한 첫 프로젝트였습니다. 만족스러운 결과물이 나오진 못했지만 , 내가 보완해야 할 문제점들을 파악할 수 있어 배운점이 많았습니다.

  • # 다양한 크기와 비율

    퍼블리싱을 할때 mobile, tablet, pc 에서 고려해야할 크기가 다 달라 적용시키기 어려웠습니다. gird를 통해 content를 정렬 시킨 후 가변 단위를 이용해 유연한 반응형 웹사이트를 만들었습니다.

  • # 플러그인 사용

    처음에는 비디오 삽입을 iframe 방식으로 적용시켰습니다. 보안 문제로 jQueyr 플러그인을 사용했습니다. jQuery 플러그인 이용하여 영상을 깔끔하게 보여주고 컨트롤 바, 공유 등 불필요한 속성들을 제어할 수 있었습니다.
